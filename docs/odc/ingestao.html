Ingestão de dados¶

Com os produtos e seus datasets catalogados, é possível realizar um processo de ingestão de dados. Neste processo, transformações como reprojeção e tiling podem ser aplicados nos dados para que eles sejam mais simples de serem carregados pelo ODC e tornem-se mais usuais para diferentes análises dos usuários.

A realização do processo de ingestão, assim como em todas as demais etapas apresentadas neste tutorial, exige um arquivo YAML. Para este caso, o stac2odc ainda não tem utilitários disponíveis. Para contornar esta questão e tornar fácil o processo de ingestão, utilize o arquivo CB4_64_16D_STK_v1.ingest.yaml mostrado abaixo, que contém a descrição desta operação:

Trecho de Código 1 - Arquivo com a descrição do processo de ingestão do produto CB4_64_16D_STK_v1 (CB4_64_16D_STK_v1.ingest.yaml).¶
source_type: CB4_64_16D_STK_v1
output_type: CB4_64_16D_STK_v1_ingested

description: CBERS-4 ingested data

location: '/data/ingested/'
file_path_template: 'CB4_64_16D_STK_v1_ingested/CB4_64_16D_STK_v1_ingested_{tile_index[0]}_{tile_index[1]}_{start_time}.nc'

global_attributes:
    title: CBERS 4 ingested
    summary: CBERS-4 data product
    source: CBERS 4 version 1
    institution: INPE
    instrument: AWFI
    cdm_data_type: Grid
    keywords: REFLECTANCE,CBERS,EARTH SCIENCE
    platform: CBERS-4
    processing_level: L2
    product_version: '1.0'
    product_suite: INPE CBERS4
    project: BDC
    naming_authority: bdc.inpe
    acknowledgment: CBERS4 is provided by the National Institute for Space Research (INPE).

storage:
    driver: NetCDF CF

    crs: +proj=aea +lat_0=-12 +lon_0=-54 +lat_1=-2 +lat_2=-22 +x_0=5000000 +y_0=10000000 +ellps=GRS80 +units=m +no_defs
    tile_size:
            x: 100000.0
            y: 100000.0
    resolution:
            x: 64
            y: -64
    chunking:
        x: 200
        y: 200
        time: 1
    dimension_order: ['time', 'y', 'x']

measurements:
    - name: blue
      dtype: int16
      nodata: -9999
      resampling_method: nearest
      src_varname: 'blue'
      zlib: True
      attrs:
        long_name: "BAND13"
        alias: "BAND13"

    - name: evi
      dtype: int16
      nodata: -9999
      resampling_method: nearest
      src_varname: 'evi'
      zlib: True
      attrs:
        long_name: "EVI"
        alias: "EVI"

    - name: green
      dtype: int16
      nodata: -9999
      resampling_method: nearest
      src_varname: 'evi'
      zlib: True
      attrs:
        long_name: "BAND14"
        alias: "BAND14"

    - name: ndvi
      dtype: int16
      nodata: -9999
      resampling_method: nearest
      src_varname: 'ndvi'
      zlib: True
      attrs:
        long_name: "BAND16"
        alias: "BAND16"

    - name: red
      dtype: int16
      nodata: -9999
      resampling_method: nearest
      src_varname: 'red'
      zlib: True
      attrs:
        long_name: "BAND15"
        alias: "BAND15"

O processo de ingestão presente no arquivo CB4_64_16D_STK_v1.ingest.yaml, realiza a compressão dos dados e aplica tiling, isso para que a recuperação dos dados seja feita de forma mais rápida. A geração da ingestão é apresentada abaixo:

datacube -v ingest \
         -c CB4_64_16D_STK_v1.ingest.yaml \
         --executor multiproc 2

Além da opção --executor, aplicada no comando acima, existem diversas outras que podem ser utilizadas para acelerar o processo de ingestão e também de garantir sua reprodutibilidade. Para mais informações consulte a documentação do ODC sobre ingestão.
